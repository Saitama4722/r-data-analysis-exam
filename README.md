# Экзамен по анализу данных в R

Проект содержит решение экзаменационного задания по анализу данных: один воспроизводимый скрипт `solution.R` выполняет шесть блоков задач — от загрузки опросника Big Five до визуализации временного ряда и анимации. Все таблицы и графики сохраняются в папку `outputs`.

## Данные и задачи

**Локальные данные:** файл `data_5.csv` — ответы респондентов по пунктам опросника Big Five. Формат CSV с разделителем «;» и значениями в кавычках (экспорт из Excel с европейской локалью). Столбцы: идентификатор строки и по 10 пунктов для каждой из пяти шкал (EXT1–EXT10, EST1–EST10, AGR1–AGR10, CSN1–CSN10, OPN1–OPN10); допустимые значения пунктов — 1–5.

**Внешние источники:** рейтинги и описания настольных игр загружаются по URL датасета TidyTuesday (ratings и details); месячные данные по числу рождений в Нью-Йорке (1946–1959) — с сайта Rob J. Hyndman.

Скрипт последовательно выполняет:

1. **Задание 1** — чтение `data_5.csv`, осмотр структуры и объёма, подсчёт пропусков и нулей в столбцах шкал. Сравнение двух вариантов обработки нулей (как обычные значения и как пропуски NA); выбор варианта по качеству k-means кластеризации при k=3 (средняя ширина silhouette) и по отсутствию искусственно мелких кластеров.

2. **Задание 2** — расчёт по каждому респонденту средних по шкалам EXT, EST, AGR, CSN, OPN (с учётом выбранной обработки нулей), сводные средние по выборке, столбчатая диаграмма средних по шкалам и ящики с усами по распределению шкал.

3. **Задание 4** — кластеризация респондентов по стандартизованным шкалам Big Five: подбор числа кластеров по методу локтя (WSS) и по средней ширине silhouette, финальная модель k-means с k=3, присвоение меток кластеров, расчёт и сохранение профилей (средние по шкалам в каждом кластере) и визуализация профилей.

4. **Задание 3** — объединение датасетов настольных игр по `id`, формирование переменной «число игроков» (среднее от min и max), отбор полных случаев по рейтингу и предикторам. Корреляции среднего рейтинга с временем игры, числом игроков, годом издания и минимальным возрастом; диаграммы рассеяния с линией линейного тренда; линейная регрессия рейтинга по этим предикторам. Сохранение объединённой таблицы и таблицы коэффициентов модели.

5. **Задание 6** — загрузка месячной серии рождений в Нью-Йорке, построение временного ряда, статичный график и анимация раскрытия ряда по дате (GIF через gganimate и gifski).

## Структура проекта

| Файл / папка | Назначение |
|--------------|------------|
| `solution.R` | Основной скрипт: установка пакетов, задания 1–6, итоговое резюме в консоль. Запускать из каталога, где лежит `data_5.csv`. |
| `data_5.csv` | Входные данные опросника Big Five (разделитель «;», кавычки). |
| `outputs/` | Каталог результатов (создаётся скриптом при отсутствии). |

**Файлы в `outputs/`:** скрипт записывает таблицы в CSV и графики в PNG/GIF. После полного прогона в папке появляются:

- **Задание 2:** `task2_bar_scale_means.png`, `task2_boxplot_scales.png`
- **Задание 4:** `task4_elbow.png`, `task4_silhouette.png`, `task4_clusters_bars.png`, `task4_cluster_means.csv`
- **Задание 3:** `task3_scatter_playingtime.png`, `task3_scatter_players.png`, `task3_scatter_yearpublished.png`, `task3_scatter_minage.png`, `boardgames_merged.csv`, `boardgames_model_coefficients.csv`
- **Задание 6:** `task6_timeseries.png`, `task6_births_animation.gif` (GIF при успешном рендере gifski)

В репозитории могут присутствовать только часть файлов (например, только CSV); остальные создаются при запуске `solution.R`.

## Результаты

**Кластеризация (задание 4).** По таблице `task4_cluster_means.csv`: три кластера с размерами 328, 275 и 406 респондентов. Второй кластер в среднем выше по экстраверсии, доброжелательности, добросовестности и открытости; первый выделяется более высокой эмоциональной стабильностью при средних остальных показателях; третий — с наиболее низкими значениями по эмоциональной стабильности и добросовестности. Явного типа «всё высокое» или «всё низкое» нет: у каждой группы свой профиль по шкалам.

**Настольные игры (задание 3).** Объединённый датасет в `boardgames_merged.csv`; коэффициенты линейной модели в `boardgames_model_coefficients.csv`. С рейтингом слабее всего связаны время игры и число игроков; заметнее — год издания и минимальный возраст: более новые и «взрослые» игры в среднем получают чуть более высокий рейтинг. Корреляции и регрессия отражают совместную изменчивость, а не причинно-следственные связи.

**Временной ряд (задание 6).** Статичный график и анимация показывают месячную динамику рождений в Нью-Йорке в 1946–1959 годах с выраженной сезонностью и трендом.

## Запуск

1. Убедитесь, что в одной папке лежат `solution.R` и `data_5.csv`.
2. Откройте `solution.R` в R или RStudio и выполните весь скрипт сверху вниз (например, через «Source»).
3. Для заданий 3 и 6 нужен доступ в интернет (загрузка данных TidyTuesday и серии рождений).
4. При первом запуске скрипт установит недостающие пакеты из CRAN: `readr`, `ggplot2`, `tidyr`, `dplyr`, `cluster`, `gganimate`, `gifski`.

## Используемые пакеты

- **readr** — чтение CSV (в том числе по URL)
- **ggplot2** — все статичные графики
- **tidyr** — преобразование в длинный формат (pivot_longer)
- **dplyr** — фильтрация, группировка, сводки, объединение таблиц
- **cluster** — расчёт silhouette для кластеризации
- **gganimate**, **gifski** — анимация временного ряда и сохранение в GIF
